### 关键技术点

**1. MyBatis CALLABLE + OUT 参数回填**
- 用 `Map<String, Object>` 同时传 IN 参数和接收 OUT 参数
- `mode=OUT` 的键在执行后自动写入 Map
- `statementType="CALLABLE"` + `parameterType="map"`

**2. 购票流程**
```
前端 JSON → TicketRequest DTO → Service 加密身份证(AESUtil) 
→ 调 sp_book_ticket(11个IN + 3个OUT) → Map 回填 
→ resultCode==0 → Result.success({orderId, resultMsg})
→ resultCode!=0 → Result.error(resultMsg)
```

### 表关系理解

```
查车次：station → train_route ← train → train_schedule   (4表, 只读)
查座位：carriage → seat  │  seat_inventory (SP内部查冲突)   (3表, 只读)
购票：  ticket_order + seat_inventory  (SP内部写, Java不碰)
```

**核心教训：12 张表里 Java 真正直接查询的只有 station/train/carriage/seat/passenger 5 张，其余全是 SP + 触发器幕后干活。**
